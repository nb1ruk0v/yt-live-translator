# Subtitles: убрать line-count cap, ширина 64

## Контекст

`src/subtitles.py:_wrap` сейчас режет переводы, не влезающие в бюджет
`width × max_lines`, добавляя `…`. На реальном файле
`data/Prompting for Agents_dubbed.ru.srt` это происходит регулярно
(сегменты 8, 9, 10 в первой минуте теряют хвост).

Пред-история: бюджет уже расширяли с `42×2` до `52×3` — не помогло,
русский перевод длинного английского сегмента всё равно вылезает.

Ключевое решение: в этом проекте `.srt` — диагностический инструмент
(чтение в редакторе, grep, диф между запусками), а не конечный продукт.
Видимая обрезка содержательного текста хуже, чем длинный сабтайтл,
который может вылезти за край кадра в плеере.

## Решение

Убрать `max_lines`-cap и `…`-truncation. Оставить перенос строк по
ширине 64 символа — чтобы файл оставался читаемым в редакторе и не
превращался в одну гигантскую строку на cue.

Ничто другое в пайплайне не меняется: дубляж, тайминги, перевод,
группировка остаются как есть.

## Изменения

### `src/subtitles.py`

`_wrap` упрощается:

```python
def _wrap(text: str, width: int = 64) -> str:
    """Wrap text into lines of at most `width` chars. No line-count cap."""
    return "\n".join(textwrap.wrap(text, width=width))
```

Параметр `max_lines` удаляется. Логика обрезки и добавления `…` —
тоже. Docstring обновляется.

Вызов в `write_srt` остаётся прежним: `text = _wrap(_escape(raw))`.

### `tests/test_subtitles.py`

Удаляются тесты, проверяющие исчезающее поведение:

- `test_overflow_truncates_with_ellipsis`
- `test_exact_two_lines_no_ellipsis` (тестировал «не появляется `…`,
  если влезает» — после удаления `…` тест бессмыслен)

Корректируется:

- `test_long_text_two_lines` → ширина 64, проверка `len(line) ≤ 64`
  и `≥ 2` строк.

Добавляется один новый тест:

- `test_very_long_text_no_truncation` — текст ~500 символов
  разбивается на N строк (N ≥ 5), все ≤ 64 симв, ни одна не
  заканчивается на `…`.

Тесты на `_format_timecode`, `_escape`, `write_srt` (schema, скип
пустых, нумерация) — не трогаются.

## Trade-offs

| Принимаем | Цена |
|---|---|
| Длинная фраза в плеере может вылезти за край кадра | Сабы — для дебага, не для просмотра. Если потом понадобятся «продакшн»-сабы, это будет отдельная задача. |
| Многострочные cue (5–7 строк на длинном сегменте) | Читается в редакторе нормально; в плеере перекрывает кадр — тот же аргумент. |

## Что НЕ входит

- Перевод и его длина — `translate.py` не трогается.
- Группировка сегментов (`group.py`) — не трогается.
- Sentence-aware split в SRT — отбросили в брейнсторме.
- Sync дубляжа / atempo — никак не связано.

## Verification

1. `uv run pytest tests/test_subtitles.py` — проходит.
2. На уже сгенерированном файле `data/Prompting for Agents_dubbed.ru.srt`
   после повторного прогона: `grep '…' *.srt` — пусто.
3. Открыть `.srt` в редакторе, убедиться, что бывшие усечённые
   сегменты (8, 9, 10) теперь содержат полный перевод.
